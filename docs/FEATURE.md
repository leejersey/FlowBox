##  第一层：打磨现有体验（1-2 天可落地）

### 1\. 🧲 跨模块智能关联

目前待办、灵感、语音、剪贴板是各自独立的孤岛。可以构建一个**轻量关系图谱**：

*   语音转写自动提取的待办，卡片上带有"🎙️ 来源：语音 #12"的可点击回溯链接
*   灵感卡片可以"关联到待办"，在待办详情页看到相关灵感
*   **统一搜索**：一个全局搜索框，跨模块返回结果（待办 / 灵感 / 语音 / 剪贴板）

### 2\. 📌 智能剪贴板增强——"剪贴板工作流"

不只是存，还要能**串联使用**：

*   **剪贴板模板**：定义常用的文本模板（如邮件模板、代码片段），一键填充变量后复制
*   **剪贴板历史 Diff**：当同一来源的内容被多次复制，自动高亮差异（类似 Git Diff）
*   **批量拼接**：勾选多条剪贴板记录，自动拼接成一段文本复制出去

### 3\. ⏱️ 番茄钟 + 白噪音

番茄专注页加入内置的**环境音效**（雨声、咖啡馆、壁炉），使用 Web Audio API 播放。不需要后端，几十 KB 的音频循环即可，但体验升级感极强。

## 🚀 第二层：有辨识度的差异化功能（3-5 天）

### 4\. 🧠 AI 每日回顾 / 晚日记

每天晚上 9 点（或用户自定义），AI Butler **主动弹窗**生成一份"今日回顾"：

📋 今日完成 3 项待办（完成率 75%）

💡 捕获 2 条灵感

🍅 专注 4 个番茄（共 100 分钟）

🎙️ 录制 1 段语音备忘

🤔 AI 建议：你有 2 条高优先级待办连续 3 天没处理，

   要不要明天优先攻克？

这个功能只需要聚合 `statsService` 现有数据 + 一次 AI 总结调用即可，但会让用户产生**习惯性打开 FlowBox 的冲动**。

### 5\. 📸 截图 OCR → 灵感 / 待办

利用 Tauri 的原生能力，注册一个**全局截图快捷键**（如 `Cmd+Shift+S`）：

*   截取屏幕区域
*   通过 AI Vision（OpenAI GPT-4o）或本地 OCR 识别文字
*   弹出一个浮窗问用户："识别到以下内容，要存为灵感还是创建待办？"
*   这是一个非常有"效率工具"辨识度的功能

### 6\. 🗂️ "知识卡片"视图（灵感升级）

灵感模块从"速记本"升级为**个人知识库**：

*   支持灵感之间的**双向链接**（类似 Obsidian 的 `[[]]` 语法）
*   增加一个"图谱视图"（用 D3.js 或 react-force-graph），可视化灵感之间的关联
*   AI 自动推荐："这条灵感可能和 3 天前的 XX 相关"

## 💎 第三层：大胆探索（v0.3+ 里程碑级别）

### 7\. 🔄 "Flow 模式"——沉浸式工作流引擎

FlowBox 的名字里就有 "Flow"，为什么不做一个真正的**心流模式**？

*   用户声明一个任务目标（如"写完项目报告"）
*   进入 Flow 模式后：
    *   自动启动番茄钟
    *   屏蔽外部通知
    *   监测应用使用——如果用户切到微信/B站超过 30 秒，温柔弹窗提醒"你偏离了目标 🤔"
    *   结束后自动生成一份"Flow 报告"：专注时长、中断次数、完成的待办

### 8\. 🌊 情绪追踪 + AI 洞察

在每日回顾时增加一个**情绪打分**（1-5 星 / emoji 选择），长期积累后：

*   AI 发现模式："你在周三通常效率最高，周五容易低落"
*   关联分析："你专注时间超过 2 小时的日子，情绪评分平均高 23%"
*   这是目前市面上效率工具很少触及的维度

### 9\. 🤝 轻量协作——"FlowBox Share"

保持 Local-First 的前提下，增加**单向分享**能力：

*   将一条灵感/一份语音转写/一份 AI 周报导出为精美的**静态网页**（单个 HTML 文件）
*   通过 AirDrop / 邮件 / 微信发送
*   不需要服务器，不破坏隐私，但让 FlowBox 的内容有了社交传播力

### 10\. 🧩 插件系统（长期愿景）

参考 Obsidian/Raycast 的模式，让用户或开发者编写轻量插件：

*   自定义 Butler 快捷指令
*   新增数据源（如连接 GitHub Issues、读取 RSS）
*   自定义效率看板卡片
